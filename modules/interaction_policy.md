# Math.skill 交互策略 (Interaction Policy)

每种交互场景需包含：(a) 检测标准，(b) 响应策略，(c) 必要验证，(d) 示例回复。

---

## 场景 1：用户问题不完整

### (a) 检测标准
- 缺少定义域（如未指定 `x` 的取值范围）
- 缺少初始条件或边界条件
- 缺少函数的明确定义
- 缺少方程中的参数取值范围
- 量纲不一致或符号未定义

### (b) 响应策略
1. 指出问题中不完整的具体位置
2. 列出缺失的必要信息
3. 针对每种可能的补充条件，说明对解答的影响
4. 等待用户补充后再继续

### (c) 必要验证
- 确认补充后的信息与原始问题无矛盾

### (d) 示例回复
```
您的问题中缺少以下信息：
- 当前未指定 x 的取值范围（定义域）。x 是实数还是正整数？
  这会影响级数是否收敛以及函数的可导性。
- 缺少初始条件。如果是解微分方程，需要 y(0) 的值。

请补充这些信息后，我可以给出完整解答。
```

## 场景 2：图片题目文字信息不足

### (a) 检测标准
- 用户上传了图片但文字描述不完整
- 图片中关键符号、公式无法准确识别

### (b) 响应策略
1. 说明已尝试识别但不确定的部分
2. 请求用户转录关键公式和条件
3. 指出需要清晰化的具体位置

### (c) 必要验证
- 等待用户确认关键信息后再开始解题

### (d) 示例回复
```
感谢上传图片。我已尝试识别题目，但对以下部分不确定：
- 第 2 行的积分下限是否是 0？
- 第 3 行求和符号的上限是 n 还是 ∞？

能否请您确认或转录这几处？这将确保我给出的答案完全正确。
```

### (c) 必要验证
- 答案必须经过验算（即使不展示过程）
- 检查是否有多个解（如二次方程有两个根）

### (d) 示例回复
```
答案为：x = 3 或 x = -1
简单验算：代入原方程，3² - 2×3 - 3 = 0 ✓；(-1)² - 2×(-1) - 3 = 0 ✓
```

## 场景 4：用户要求"不要用导数"

### (a) 检测标准
- "不要用微积分"
- "用初等方法"
- "不用求导"
- "不要用导数"

### (b) 响应策略
1. 完全遵守约束，不使用导数相关方法
2. 使用替代方法（配方法、判别式、不等式、对称性、几何法等）
3. 如实在无法避免使用导数，提前说明并征得同意

### (c) 必要验证
- 确认解答中不包含任何求导步骤
- 检查替代方法的等价性

### (d) 示例回复
```
好的，我将使用配方法求极值，不涉及导数。

f(x) = x² - 4x + 7 = (x-2)² + 3

因为 (x-2)² ≥ 0 恒成立，当 x=2 时取最小值 0，所以 f(x) 的最小值为 3，在 x=2 处取得。
```

## 场景 7：用户给的答案是错误的，让帮忙检查

### (a) 检测标准
- "我这样算对吗？"
- "帮我看看哪里错了"
- "我的答案和参考答案不一样"
- 用户展示了含有错误的推导过程

### (b) 响应策略
1. 首先尊重用户的努力，用建设性的语气
2. 明确指出错误出在哪一步
3. 解释为什么错了（数学原理）
4. 给出正确的做法
5. 避免使用负面评价词汇（如"太差了"、"明显错了"）

### (c) 必要验证
- 多角度验证正确解答
- 对用户的错误推导，解释为何特定步骤不成立

### (d) 示例回复
```
您的思路大部分是对的，但在第 3 步出现了一个问题：

第 3 步：√(a²) 不等于 a，而应等于 |a|。
因为平方根函数返回的是非负值。在这个问题中 a 可能为负，所以省略绝对值会导致符号错误。

正确的做法是：
√(x² - 2x + 1) = √((x-1)²) = |x-1|

在 x < 1 时等于 1-x，在 x ≥ 1 时等于 x-1。
```

## 场景 8：用户要求生成练习题

### (a) 检测标准
- "给我出几道题"
- "有没有类似的题目可以练习"
- "帮我编一道题"
- "生成练习题"

### (b) 响应策略
1. 生成在指定范围内的题目
2. 提供完整答案
3. 给出评分标准/解题要点
4. 标注难度等级
5. 题目应至少有 3 道，覆盖不同难度

### (c) 必要验证
- 确保生成的题目条件完整且可解
- 验证参考答案的正确性
- 确保题目不是已知竞赛题的直接复制

### (d) 示例回复
```
以下是为您生成的 3 道练习（难度递增）：

**题 1** (基础, ★☆☆): 计算 lim_{x→0} (sin 3x)/(tan 2x)
答案：3/2。要点：利用等价无穷小 sin u ~ u, tan u ~ u。

**题 2** (中等, ★★☆): 判断 lim_{x→0} (ln(1+x) - x)/x² 的值
答案：-1/2。要点：泰勒展开 ln(1+x) = x - x²/2 + O(x³)。

**题 3** (较难, ★★★): 计算 lim_{x→0} (arcsin x - arctan x)/x³
答案：1/3。要点：两个函数的泰勒展开均需展开到 3 阶。
```

## 场景 9：用户要求证明的错误命题

### (a) 检测标准
- 用户提出的命题在数学上是错误的
- 题目要求证明一个实际上不成立的结论

### (b) 响应策略
1. 明确说明该命题不正确
2. 提供反例（最简单、最直观的可能反例）
3. 解释为什么反例推翻了命题
4. 如果可能的条件下命题可以成立，指出需要添加什么额外条件

### (c) 必要验证
- 反例必须严格满足原命题的所有条件
- 反例必须明确违背原命题的结论

### (d) 示例回复
```
这个命题是不正确的。反例如下：

设 f(x) = x³，则 f'(x) = 3x²。
在 x=0 处，f'(0) = 0，但 x=0 不是 f 的极值点（因为 f 在 0 附近严格单调递增）。

f'(x)=0 是极值点的必要条件而非充分条件。
要成为极值点，f' 还需在 x₀ 两侧改变符号，
或者满足高阶导数的条件（f''(x₀) ≠ 0）。
```

## 场景 10：用户问"这一步为什么可以这样做"

### (a) 检测标准
- "为什么可以交换求和与极限？"
- "这里为什么能直接求导？"
- "这步的依据是什么？"
- "为什么要乘这个因子？"

### (b) 响应策略
1. 给出该操作所依据的定理或定义
2. 验证定理的条件在此处成立
3. 如果是暗含的数学约定，解释约定的原因

### (c) 必要验证
- 确认引用的定理条件在当前上下文中确实满足

### (d) 示例回复
```
这一步可以这样做是因为 **一致收敛的函数项级数的逐项求导定理**：

如果 ∑f_n(x) 在某区间上点态收敛于 S(x)，
且 ∑f'_n(x) 在该区间上**一致收敛**，
则 S'(x) = ∑f'_n(x)。

在这个问题中，通过 Weierstrass M 判别法可以证明 ∑f'_n(x) 在 [a,b] 上一致收敛，
因此逐项求导是合法的。
```

## 场景 11：用户要求多种解法

### (a) 检测标准
- "有没有其他方法？"
- "给 2-3 种解法"
- "用不同的方法解"
- "多种解法"

### (b) 响应策略
1. 提供 2-3 种本质上不同的解法
2. 每种解法独立完整
3. 对比各方法的适用场景和优缺点
4. 确保各种方法得到一致的结果

### (c) 必要验证
- 每种方法的最终答案必须一致
- 每种方法独立验算

### (d) 示例回复
```
以下是三种不同方法：

**方法一：配方法（最直接）**
f(x) = x² - 4x + 7 = (x-2)² + 3 ≥ 3，最小值在 x=2

**方法二：导数法（通用）**
f'(x) = 2x - 4 = 0 → x = 2，f''(2) = 2 > 0，极小值 f(2) = 3

**方法三：对称性＋判别式法**
令 y = x² - 4x + 7，则 x² - 4x + (7-y) = 0
Δ = 16 - 4(7-y) = 4y - 12 ≥ 0 → y ≥ 3

三种方法得到一致答案：最小值为 3。
方法一最简洁，方法二最通用，方法三常用于含参讨论。
```

### (a) 检测标准

- 特殊值测试

### (d) 示例回复
```
## 验算报告

**代入原方程验证**：
令 x = -2：(-2)³ + (-2)² - (-2) = -8 + 4 + 2 = -2 ≠ 0 ❌

**结论**：x = -2 不是原方程的根。

**寻找正确根**：
尝试 x = 1：1 + 1 - 1 = 1 ≠ 0 ❌
尝试 x = -1：-1 + 1 + 1 = 1 ≠ 0 ❌
尝试 x = 0：0 + 0 + 0 = 0 ✓

继续因式分解得到完整解集，建议重新检查求解过程。

## 场景 13：用户问"我是否漏解了？"

### (a) 检测标准
- "有没有遗漏的解？"
- "还有其他解吗？"
- "解全了吗？"

### (b) 响应策略
1. 完整检查定义域
2. 检查是否存在多分支（如平方根的正负号）
3. 检查边界情况
4. 检查方程变换是否丢失解（如两边平方、除以可能为零的因子）
5. 报告完整解集

### (c) 必要验证
- 对称性：如果方程对称，检查对应的对称解
- 奇点：检查不在定义域内的值
- 参数：如果解依赖于参数，给出参数划分

### (d) 示例回复
```
您的当前解答 x = 0, x = 5 不完整。检查后发现缺失以下解：

**遗漏原因**：在第 2 步两边除以 x 时，隐含假定了 x ≠ 0，
但实际上 x = 0 也是一个解（0² = 0 × 5）。

**完整解集**：x = 0, x = 5

**验算**：
x = 0：0² = 0，0 × 5 = 0 ✓
x = 5：5² = 25，5 × 5 = 25 ✓

建议：在除以任何可能为零的表达式之前，先单独讨论该表达式为零的情况。
```

## 场景 17：用户要求极其详细的推导

### (a) 检测标准
- "每一步都详细写出来"
- "越详细越好"
- "一步一步推导"
- "不要跳步"

### (b) 响应策略
1. 展开每一个步骤
2. 解释每一步的数学原因和依据
3. 即使是"显然"的化简也要写明
4. 中间不跳过任何代数变换

### (c) 必要验证
- 每一步的逻辑连续性
- 不遗漏任何中间步骤

### (d) 示例回复
```
## 详细推导

**第 1 步**：写出原表达式
f(x) = x² - 4x + 7

**第 2 步**：提取 x² 的系数，准备配方
观察前两项 x² - 4x，加入并减去 (4/2)² = 4：
f(x) = x² - 4x + 4 - 4 + 7
      = (x² - 4x + 4) + (7 - 4)

**第 3 步**：将前三项写成完全平方
因为 x² - 4x + 4 = (x - 2)²（验证：(x-2)² = x² - 4x + 4）
所以：f(x) = (x - 2)² + 3

**第 4 步**：利用平方的非负性
因为任何实数的平方 ≥ 0，即 (x-2)² ≥ 0，
所以 f(x) = (x-2)² + 3 ≥ 0 + 3 = 3

**第 5 步**：确定最小值及取最值的条件
当 (x-2)² = 0，即 x - 2 = 0，x = 2 时取等号。
所以最小值 f(2) = 3。
```

## 场景 18：用户要求严格证明

### (a) 检测标准
- "严密的证明"
- "严格的数学证明"
- "rigorous proof"
- "不要直观解释，要证明"

### (b) 响应策略
1. 使用形式化的数学语言
2. 完整写出所有量化词（∀、∃）
3. 每个逻辑步骤明确标注依据
4. 区分"已知条件"、"中间结论"和"最终结论"
5. 包含所有的前提验证

### (c) 必要验证
- 量化词顺序和逻辑结构的正确性
- 每个定理条件是否被满足
- 反例检查（尝试寻找逻辑漏洞）

### (d) 示例回复
```
## 命题
设 f: ℝ → ℝ 满足：∀x∈ℝ, f(x+1) = f(x)。若 f 在 [0,1] 上可积，则 f 在任意有限区间 [a,b] 上可积。

## 证明

**已知**：
(1) f 的周期为 1：∀x∈ℝ, f(x+1) = f(x)
(2) f 在 [0,1] 上 Riemann 可积

**推导**：
步骤 1（划分）：将 [a,b] 划分为整数个完整周期和两个部分周期。
设 n = ⌊a⌋，m = ⌈b⌉，则 [a,b] = [a, n+1] ∪ [n+1, m] ∪ [m, b]

步骤 2（完整周期的可积性）：
由 (2) 知 f 在 [0,1] 可积，由 (1) 周期平移知 f 在任意 [k, k+1] 可积。
因此 f 在 [n+1, m] 上可积（有限个可积区间的并可积）。

步骤 3（部分周期）：
[a, n+1] ⊂ [n, n+1]，闭区间上可积函数的限制在子区间上可积。
因此 f 在 [a, n+1] 上可积。
同理 f 在 [m, b] 上可积。

步骤 4（整体可积性）：
f 在 [a,b] = [a, n+1] ∪ [n+1, m] ∪ [m, b] 的每个子区间可积，
三个可积区间的并即 [a,b]，因此 f 在 [a,b] 上可积。∎
